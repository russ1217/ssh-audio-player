# Tab切换刷新功能测试指南

## 测试步骤

### 测试1：基本Tab切换
1. 启动应用
2. 点击底部导航栏的"播放列表"Tab
3. 再点击"文件"Tab
4. **预期结果**：文件列表正常显示，不会一直转圈

### 测试2：播放后切换
1. 启动应用
2. 在文件列表中选择一个音频文件并播放
3. 切换到"播放列表"Tab
4. 再切换回"文件"Tab
5. **预期结果**：文件列表正常显示，底部播放器继续工作

### 测试3：本地/SSH模式切换
1. 启动应用，使用本地模式浏览文件
2. 切换到SSH模式（如果已配置）
3. 播放一个SSH远程文件
4. 切换回本地模式
5. 切换到"播放列表"Tab
6. 再切换回"文件"Tab
7. **预期结果**：显示本地文件列表，不会转圈

### 测试4：多次快速切换
1. 快速在"文件"、"播放列表"、"设置"之间来回切换5-10次
2. **预期结果**：每次切换到"文件"Tab都能正常显示，不会出现状态混乱

### 测试5：手动刷新
1. 在文件浏览器界面
2. 点击右上角的刷新按钮
3. **预期结果**：当前目录重新加载，显示最新文件列表

## 常见问题

### Q1: 如果还是看到转圈怎么办？
A: 检查日志中是否有以下信息：
- `⚠️ 检测到异常的loading状态（文件已加载），强制重置` - 这表示修复正在工作
- `⚠️ 目录正在加载中，跳过重复请求` - 这表示防重复加载机制在工作

### Q2: 如何确认refreshCounter在工作？
A: 每次Tab切换时，refreshCounter应该递增。可以在代码中添加临时日志：
```dart
debugPrint('🔢 refreshCounter: ${provider.refreshCounter}');
```

### Q3: 本地模式和SSH模式切换有问题？
A: 确保：
- 本地模式有存储权限
- SSH模式已正确连接
- 切换模式后会调用 `forceRefreshCurrentDirectory`

## 性能优化建议

如果未来发现频繁刷新影响性能，可以考虑：
1. 添加 debounce 机制，避免短时间内多次刷新
2. 缓存文件列表，只在必要时重新加载
3. 使用更细粒度的状态管理，只刷新变化的部分
