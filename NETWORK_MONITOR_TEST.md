# 网络状态检测功能测试指南

## 快速测试步骤

### 准备工作
1. 确保已配置 SSH 服务器并可以正常连接
2. 准备一些音频文件用于测试播放

### 测试场景 1: 基本网络断开恢复

#### 步骤：
1. **启动应用并连接 SSH**
   ```
   - 打开应用
   - 连接到你的 SSH 服务器
   - 浏览到包含音频文件的目录
   ```

2. **开始播放音频**
   ```
   - 选择一个音频文件
   - 点击播放按钮
   - 确认音频正常播放
   ```

3. **模拟网络断开**
   ```
   方法 A: 关闭 WiFi
   - Android: 下拉通知栏，关闭 WiFi
   - 或飞行模式开启/关闭
   
   方法 B: 路由器断电
   - 拔掉路由器电源
   - 等待 5-10 秒
   ```

4. **观察应用行为**
   ```
   预期结果:
   - 播放停止（因为 SSH 断开）
   - 控制台输出: "⚠️ 网络已断开，保存播放状态..."
   - 不应出现崩溃或错误对话框
   ```

5. **恢复网络连接**
   ```
   - 重新开启 WiFi
   - 或恢复路由器供电
   - 等待网络完全连接（约 5-10 秒）
   ```

6. **验证自动恢复**
   ```
   预期结果:
   - 控制台输出: "✅ 网络已恢复，检查是否需要重连和恢复播放..."
   - 控制台输出: "🔄 网络恢复，尝试重新连接SSH..."
   - 控制台输出: "✅ SSH 重连成功，准备恢复播放"
   - 音频从断开时的位置继续播放
   - 播放进度条显示正确的位置
   ```

### 测试场景 2: 网络频繁波动

#### 步骤：
1. 开始播放音频
2. 快速开关 WiFi 2-3 次（每次间隔 3-5 秒）
3. 观察应用是否正确处理，不会重复恢复

#### 预期结果：
- 防抖机制生效，不会多次触发恢复
- 最终网络稳定后能正确恢复播放

### 测试场景 3: 本地文件模式不受影响

#### 步骤：
1. 切换到本地文件模式
2. 播放本地音频文件
3. 断开网络
4. 确认播放不受影响

#### 预期结果：
- 本地文件继续正常播放
- 不触发任何 SSH 重连逻辑

### 常见问题排查

#### 问题 1: 网络恢复后没有自动重连
**可能原因**:
- SSH 配置丢失
- 重连超时（超过 30 秒）

**解决方法**:
- 检查 `_activeSSHConfig` 是否为 null
- 查看是否有超时错误日志
- 手动重新连接 SSH 测试

#### 问题 2: 恢复播放时进度不正确
**可能原因**:
- 播放器未完全就绪就执行 seek 操作

**解决方法**:
- 检查日志中的 "恢复到进度" 消息
- 可能需要调整延迟时间（当前为 500ms）

#### 问题 3: 频繁触发恢复导致性能问题
**可能原因**:
- 防抖机制未生效

**解决方法**:
- 检查 `_isAutoResuming` 标志是否正确设置
- 查看是否有重复的恢复请求日志

## 性能指标

### 预期性能
- **网络检测延迟**: < 1 秒
- **SSH 重连时间**: 2-5 秒（取决于网络质量）
- **播放恢复总时间**: 5-10 秒
- **内存占用增加**: < 1 MB

### 资源使用
- NetworkMonitorService 使用单例模式，全局只有一个实例
- Stream 订阅在 dispose 时正确清理
- 无内存泄漏风险

## 已知限制

1. **仅支持 SSH 远程模式**
   - 本地文件播放不依赖网络状态

2. **需要有效的 SSH 配置**
   - 如果配置被删除，无法自动重连

3. **极端网络波动**
   - 如果网络在 30 秒内反复断开/恢复，可能需要手动重连

4. **后台播放**
   - 应用在后台时，网络检测仍然工作
   - 但重连可能受系统省电策略影响

## 后续优化建议

1. **用户提示**
   - 在网络断开/恢复时显示 Toast 通知
   - 让用户知道发生了什么

2. **重试策略优化**
   - 根据网络质量动态调整重试间隔
   - 指数退避算法

3. **离线缓存**
   - 网络断开时使用本地缓存继续播放
   - 网络恢复后同步进度

4. **统计信息**
   - 记录网络断开次数
   - 统计自动恢复成功率
