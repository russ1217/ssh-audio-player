# 播放列表存盘功能优化说明

## 功能概述

本次更新增强了播放列表的保存和加载功能，现在可以完整保存 SSH 服务器信息，并在重新加载时自动连接服务器。

## 主要改进

### 1. SSH 信息存储

**保存内容：**
- SSH 配置 ID（关联到具体的服务器配置）
- SSH 配置快照：
  - 主机地址 (host)
  - 端口号 (port)
  - 用户名 (username)
  - ⚠️ **注意：密码不会保存**（出于安全考虑）

**数据模型：**
```dart
class Playlist {
  final String? sshConfigId;              // SSH 配置 ID
  final Map<String, dynamic>? sshConfigSnapshot; // SSH 配置快照
  // ... 其他字段
}
```

### 2. UI 界面优化

#### 播放列表页面（双标签设计）

**标签 1：当前播放**
- 显示当前内存中的播放列表
- 支持拖拽排序
- 点击歌曲直接播放
- 右键菜单：播放、移除
- 顶部控制：上一曲、下一曲
- 保存按钮：将当前列表保存到数据库

**标签 2：已保存**
- 显示所有历史保存的播放列表
- 每个列表卡片展示：
  - 📋 播放列表名称
  - 🎵 歌曲数量
  - ☁️ SSH 服务器信息（用户名@主机）
  - 🕒 上次播放时间（智能格式化）
- 操作菜单：
  - ▶️ **加载并播放**：自动连接 SSH + 加载列表 + 播放第一首
  - 👁️ **查看内容**：浏览列表中的所有歌曲
  - ✏️ **重命名**：修改播放列表名称
  - 🗑️ **删除**：确认后删除列表

### 3. 自动连接流程

当用户点击"加载并播放"时：

```
1. 显示加载对话框 "正在连接 SSH 并加载播放列表..."
   ↓
2. 检查当前 SSH 连接状态
   ├─ 如果未连接 → 使用保存的配置重新连接
   └─ 如果已连接到不同服务器 → 切换到目标服务器
   ↓
3. 从数据库读取播放列表
   ↓
4. 将列表项转换为 MediaFile 对象
   ↓
5. 加载到内存播放列表
   ↓
6. 关闭加载对话框
   ↓
7. 显示成功提示 "已加载播放列表: XXX"
   ↓
8. 自动播放第一首歌曲
```

**错误处理：**
- 如果 SSH 连接失败 → 显示错误提示
- 如果找不到 SSH 配置 → 提示用户先添加服务器
- 如果文件不存在 → 跳过该文件继续播放下一首

### 4. 使用示例

#### 场景 1：保存当前播放列表

1. 在"文件"标签页浏览远程服务器文件
2. 点击多个音频文件添加到播放列表
3. 切换到"播放列表"标签页
4. 点击右上角 💾 保存图标
5. 输入名称，例如："古典音乐精选"
6. 点击"保存"

✅ 系统自动保存：
- 所有歌曲的路径和文件名
- 当前 SSH 服务器的配置快照
- 保存时间戳

#### 场景 2：加载已保存的播放列表

1. 切换到"播放列表" → "已保存"标签
2. 找到之前保存的"古典音乐精选"
3. 点击卡片或选择"加载并播放"
4. 如果 SSH 未连接，会提示输入密码
5. 连接成功后自动开始播放第一首

✅ 体验流畅，无需手动：
- 重新输入服务器地址
- 重新浏览文件夹
- 逐个添加歌曲

#### 场景 3：管理多个播放列表

可以为不同的场景创建多个播放列表：
- 🎸 "摇滚经典" - 连接到服务器 A
- 🎹 "钢琴曲集" - 连接到服务器 B  
- 🎤 "KTV 歌单" - 连接到服务器 C

每个列表都独立保存了对应的 SSH 配置，切换时自动连接正确的服务器。

## 技术实现细节

### 数据持久化

使用 `SharedPreferences` 存储：
```dart
// 播放列表 JSON 结构
{
  "id": "1234567890",
  "name": "我的最爱",

  "sshConfigId": "ssh-uuid-123",
  "sshConfigSnapshot": {
    "id": "ssh-uuid-123",
    "host": "192.168.1.100",
    "port": 22,
    "username": "russ"
  },
  "items": [
    {
      "sshConfigId": "ssh-uuid-123",
      "filePath": "/music/rock/song1.mp3",
      "fileName": "song1.mp3",
      "addedAt": "2024-01-01T12:00:00.000Z"
    }
  ],
  "lastPlayed": "2024-01-02T15:30:00.000Z"

```

### 安全性考虑

**不保存的信息：**
- ❌ SSH 密码（用户需重新输入）
- ❌ 私钥文件内容

**建议的安全实践：**
1. 使用 SSH 密钥认证代替密码
2. 定期清理不再使用的播放列表
3. 避免在公共设备上保存敏感服务器的播放列表

### 性能优化

- ✅ 懒加载：只在切换到"已保存"标签时才查询数据库
- ✅ 异步操作：所有 I/O 操作不阻塞 UI
- ✅ 智能日期格式化：提升用户体验
- ✅ 加载提示：避免用户误以为应用卡死

## 已知限制

1. **密码需要重新输入**
   - 原因：安全考虑，不存储明文密码
   - 解决方案：使用 SSH 密钥认证

2. **文件路径依赖 SSH 连接**
   - 如果服务器文件被移动或删除，播放会失败
   - 建议：定期同步播放列表与实际文件

3. **不支持跨设备同步**
   - 播放列表仅保存在本地设备
   - 未来可考虑添加云同步功能

## 常见问题

**Q: 为什么加载播放列表时还要输入密码？**
A: 为了安全，我们不保存 SSH 密码。建议使用 SSH 密钥认证实现免密登录。

**Q: 可以修改已保存列表的歌曲吗？**
A: 当前版本暂不支持编辑已保存的列表。如需修改，请删除后重新保存。

**Q: 播放列表占用多少存储空间？**
A: 非常小。一个包含 100 首歌曲的列表大约只占用几 KB。

**Q: 如果服务器 IP 变了怎么办？**
A: 需要在"设置"中更新 SSH 配置，然后重新保存播放列表。
