# 清除缓存功能改进说明

## 功能概述

本次更新大幅增强了清除缓存功能，现在能够彻底清理临时目录中的所有历史文件，而不仅仅是当前追踪的缓存文件。

## 主要改进

### 1. **全面扫描临时目录**

**改进前：**
- ❌ 只清除 `_downloadCache` 中记录的文件
- ❌ 可能遗漏未被追踪的临时文件
- ❌ 无法清理应用重启前的历史文件

**改进后：**
- ✅ 扫描整个临时目录 (`getTemporaryDirectory()`)
- ✅ 识别所有音频格式的临时文件
- ✅ 递归扫描子目录
- ✅ 清理所有历史遗留文件

### 2. **智能文件识别**

系统会删除符合以下模式的文件：

**文件名模式：**
- `temp_*.mp3` - 临时 MP3 文件
- `temp_*.mp4` - 临时 MP4 视频/音频
- `temp_*.wav` - 临时 WAV 文件
- `temp_*.flac` - 临时 FLAC 文件
- `temp_*.aac` - 临时 AAC 文件
- `temp_*.m4a` - 临时 M4A 文件

**扩展名匹配：**
即使不以 `temp_` 开头，只要是常见音频格式也会被清理：
- `.mp3`, `.mp4`, `.wav`, `.flac`, `.aac`, `.m4a`

### 3. **详细的统计信息**

清除完成后会显示：
```
✅ 缓存清除完成: 删除 15 个文件，释放 2.3 GB 空间
```

包括：
- 📊 删除的文件数量
- 💾 释放的磁盘空间（自动格式化单位）
- 📝 每个文件的详细日志（调试模式）

## 技术实现

### AppProvider.clearDownloadCache()

```dart
Future<void> clearDownloadCache() async {
  // 1. 停止预下载
  _stopPredownloading();
  
  int deletedCount = 0;
  int totalSize = 0;
  
  // 2. 清除已追踪的缓存文件
  for (final localPath in _downloadCache.values) {
    final file = File(localPath);
    if (await file.exists()) {
      totalSize += await file.length();
      await file.delete();
      deletedCount++;
    }
  }
  _downloadCache.clear();
  
  // 3. 扫描临时目录中的所有文件
  final tempDir = await getTemporaryDirectory();
  final entities = await tempDir.list().toList();
  
  for (final entity in entities) {
    if (entity is File) {
      final fileName = entity.uri.pathSegments.last;
      // 检查是否为音频文件
      if (fileName.startsWith('temp_') || 
          fileName.endsWith('.mp3') || 
          fileName.endsWith('.mp4') || 
          ...) {
        final fileSize = await entity.length();
        totalSize += fileSize;
        await entity.delete();
        deletedCount++;
      }
    } else if (entity is Directory) {
      // 递归扫描子目录
      ...
    }
  }
  
  // 4. 输出统计信息
  debugPrint('✅ 缓存清除完成: 删除 $deletedCount 个文件，释放 ${_formatFileSize(totalSize)} 空间');
}
```

### 文件大小格式化

```dart
String _formatFileSize(int bytes) {
  if (bytes < 1024) return '$bytes B';
  if (bytes < 1024 * 1024) return '${(bytes / 1024).toStringAsFixed(1)} KB';
  if (bytes < 1024 * 1024 * 1024) return '${(bytes / (1024 * 1024)).toStringAsFixed(1)} MB';
  return '${(bytes / (1024 * 1024 * 1024)).toStringAsFixed(2)} GB';
}
```

**示例输出：**
- `512 B`
- `1.5 KB`
- `23.7 MB`
- `2.34 GB`

## 使用场景

### 场景 1：定期清理释放空间

**问题：** 长期使用后，临时文件累积占用大量空间

**解决：**
1. 打开应用 → 设置
2. 查看"清除缓存"显示的当前大小
3. 点击进入，确认要清除
4. 等待清除完成
5. 看到释放了多少空间

### 场景 2：应用异常退出后的清理

**问题：** 应用崩溃或强制关闭后，临时文件可能未被正确清理

**解决：**
新的清除功能会扫描整个临时目录，即使是没有被 `_downloadCache` 追踪的文件也会被清理。

### 场景 3：版本升级后的清理

**问题：** 旧版本的临时文件格式可能与新版本不兼容

**解决：**
清除所有历史文件，确保干净的运行环境。

## 性能优化

### 1. 异步操作
- 所有文件操作都是异步的
- 不会阻塞 UI 线程
- 大目录也能流畅处理

### 2. 批量处理
- 一次性列出所有文件
- 逐个删除并统计
- 避免频繁的 I/O 调用

### 3. 异常容错
- 单个文件删除失败不影响其他文件
- 详细的错误日志便于排查问题
- 不会因为权限问题导致整个流程中断

## 安全性

### 1. 精确匹配
只删除符合特定模式的文件：
- ✅ `temp_*.mp3` - 我们的临时文件
- ✅ `*.flac` - 音频文件
- ❌ `document.pdf` - 不会被误删
- ❌ `photo.jpg` - 不会被误删

### 2. 作用域限制
- 只扫描应用的临时目录
- 不会影响系统其他位置
- 不会影响用户个人文件

### 3. 用户确认
- 清除前必须用户手动确认
- 显示将要删除的文件数量和大小
- 提供取消选项

## 常见问题

**Q: 清除缓存会影响正在播放的歌曲吗？**
A: 不会。清除前会自动停止预下载，但不会影响当前正在播放的文件。

**Q: 清除后需要重新下载所有歌曲吗？**
A: 是的，之前下载的缓存会被删除，下次播放时需要重新从服务器下载。

**Q: 为什么有些文件没有被清除？**
A: 只有符合音频文件模式（temp_ 开头或音频扩展名）的文件会被清除。其他类型的文件不受影响。

**Q: 可以定时自动清除吗？**
A: 当前版本需要手动触发。未来可以考虑添加自动清理策略（如：超过 7 天的文件自动删除）。

**Q: 清除缓存能释放多少空间？**
A: 取决于您播放过的歌曲数量和大小。在设置页面可以看到当前的缓存大小。
