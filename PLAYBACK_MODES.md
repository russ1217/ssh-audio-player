# 播放模式功能说明

## 功能概述

为SSH Audio Player添加了四种播放模式，用户可以在播放过程中随时切换：

1. **正常播放（off）**：按顺序播放，播放完最后一首后停止
2. **列表循环（all）**：按顺序播放，播放完最后一首后回到第一首继续循环
3. **单曲循环（one）**：重复播放当前歌曲
4. **随机播放（shuffle）**：随机选择下一首歌曲播放

## 技术实现

### 1. 数据模型

#### `lib/models/playlist_repeat_mode.dart`
```dart
enum PlaylistRepeatMode {
  off,      // 正常播放
  all,      // 列表循环
  one,      // 单曲循环
  shuffle,  // 随机播放
}
```

**注意**：使用`PlaylistRepeatMode`而非Flutter内置的`RepeatMode`，避免命名冲突。

### 2. 状态管理

在`AppProvider`中添加：
- `_repeatMode`: 当前播放模式
- `_shuffleIndices`: 随机播放时的索引映射列表

### 3. 核心方法

#### `toggleRepeatMode()`
循环切换播放模式：off → all → one → shuffle → off

#### `_generateShuffleIndices()`
使用Fisher-Yates洗牌算法生成随机播放顺序

#### `_getNextIndex()` / `_getPreviousIndex()`
根据当前播放模式计算下一个/上一个播放索引

### 4. UI实现

在底部播放控制栏添加播放模式切换按钮：
- **位置**：下一曲和停止按钮之间
- **图标**：
  - 正常播放：`Icons.repeat_one_outlined`（灰色）
  - 列表循环：`Icons.repeat`（白色）
  - 单曲循环：`Icons.repeat_one`（白色）
  - 随机播放：`Icons.shuffle`（白色）
- **颜色**：未激活时灰色，激活时白色（适配深色背景）
- **提示**：显示当前模式名称

## 兼容性保证

### ✅ 不影响现有功能

1. **断网重播**：播放模式切换不影响网络断开后的恢复逻辑
   - 断网恢复时使用`_resumePlaybackAfterReconnect()`，与播放模式无关
   - 恢复后保持原有播放模式

2. **暂停/播放**：`togglePlayPause()`不受播放模式影响
   - 暂停状态独立于播放模式
   - 恢复播放时从当前位置继续

3. **上一曲/下一曲**：已重构以支持不同播放模式
   - 正常模式：简单的索引递增/递减
   - 列表循环：到达边界时循环
   - 单曲循环：始终返回当前索引
   - 随机播放：使用预生成的随机索引列表

4. **文件播放完成**：`_onFileComplete()`调用`playNextInPlaylist()`
   - 自动根据当前模式选择下一首
   - 单曲循环时重新播放当前文件

## 用户使用指南

### 如何切换播放模式

1. 在底部播放控制栏找到播放模式按钮（位于下一曲和停止按钮之间）
2. 点击按钮即可切换模式
3. 按钮图标和颜色会显示当前模式：
   - 🔘 灰色圆圈 = 正常播放
   - 🔁 白色循环箭头 = 列表循环
   - 🔂 白色单循环箭头 = 单曲循环
   - 🔀 白色交叉箭头 = 随机播放

### 模式选择建议

- **正常播放**：适合一次性听完专辑或歌单
- **列表循环**：适合背景音乐或重复聆听
- **单曲循环**：适合练习或专注听某首歌
- **随机播放**：适合发现新歌曲或避免听觉疲劳
