# Android 电池优化配置

## 问题现象

应用在后台播放时，SSH 连接会在几分钟后断开，导致播放中断。

## 根本原因

Android 系统（Android 6.0+）的 **Doze 模式** 和 **应用待机模式** 会在设备静止且屏幕关闭时：
- 限制网络访问
- 限制 CPU 使用
- 延迟后台任务

这会导致 SSH 长连接被系统中断。

## 解决方案

### 方案 1：手动设置电池优化为"无限制"（推荐）

**步骤**：
1. 打开手机 **设置**
2. 进入 **应用** 或 **应用管理**
3. 找到并点击 **SSH 音频播放器**
4. 点击 **电池** 或 **电池使用**
5. 选择 **无限制** 或 **不受限制**

**不同品牌手机的路径可能不同**：

- **小米/Redmi**：设置 > 应用设置 > 应用管理 > SSH 音频播放器 > 省电策略 > 无限制
- **华为/荣耀**：设置 > 电池 > 应用启动管理 > SSH 音频播放器 > 关闭自动管理 > 允许后台活动
- **OPPO/Realme**：设置 > 电池 > 更多电池设置 > SSH 音频播放器 > 允许完全后台行为
- **vivo**：设置 > 电池 > 后台耗电管理 > SSH 音频播放器 > 允许完全后台行为
- **三星**：设置 > 电池和设备维护 > 电池 > 后台使用限制 > SSH 音频播放器 > 不受限制
- **原生 Android**：设置 > 应用 > SSH 音频播放器 > 电池 > 无限制

### 方案 2：将应用加入白名单

部分手机有"白名单"或"受保护应用"功能：
- **小米**：安全中心 > 授权管理 > 自启动管理 > 允许 SSH 音频播放器 自启动
- **华为**：手机管家 > 启动管理 > SSH 音频播放器 > 关闭自动管理
- **OPPO**：安全中心 > 权限隐私 > 自启动管理 > 允许 SSH 音频播放器

### 方案 3：保持屏幕常亮（临时方案）

在播放器设置中保持屏幕常亮，但这会增加耗电。

## 效果验证

设置完成后：
1. 播放音频文件
2. 锁定屏幕
3. 等待 5-10 分钟
4. 检查音频是否继续播放而不中断

如果仍然中断，请检查：
- 是否有其他省电软件（如绿色守护、冰箱等）限制了应用
- 是否在"睡眠模式"或"超级省电模式"下
- 网络连接是否稳定

## 开发者说明

应用已实现的自动恢复机制：
- ✅ SSH 心跳检测（60 秒正常，10 秒断开后）
- ✅ SSH 断开自动重连（最多 5 次，间隔 10 秒）
- ✅ 播放状态保存和恢复
- ✅ 流式服务 SSH 状态检测

但即使有自动恢复，电池优化仍会导致：
- SSH 连接频繁断开和重连
- 播放中断 1-2 秒（恢复期间）
- 增加不必要的网络请求

**最佳实践**：建议用户关闭电池优化，确保流畅的后台播放体验。

## 应用内提示（可选）

未来可以在应用首次启动时检测电池优化状态，并提示用户关闭：

```dart
// 检查电池优化状态
if (await isIgnoringBatteryOptimizations()) {
  // 已关闭，无需提示
} else {
  // 提示用户关闭电池优化
  showDialog(
    context: context,
    builder: (context) => AlertDialog(
      title: Text('后台播放优化'),
      content: Text('为了确保后台播放不中断，请将应用的电池优化设置为"无限制"'),
      actions: [
        TextButton(
          onPressed: () => requestIgnoreBatteryOptimizations(),
          child: Text('去设置'),
        ),
      ],
    ),
  );
}
```

需要的权限（AndroidManifest.xml）：
```xml
<uses-permission android:name="android.permission.REQUEST_IGNORE_BATTERY_OPTIMIZATIONS" />
```
