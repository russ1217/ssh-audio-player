# 全屏按钮位置说明

## 📍 全屏按钮在哪里？

### 新布局设计（两行显示）

为了在有限的空间内容纳所有控制按钮，底部播放控制栏现在采用**两行布局**：

```
┌─────────────────────────────────────┐
│  ━━━━━━━━━━━━━●━━━━━━━━━━━━━━━━━   │ ← 进度条
│  05:32 / 45:20                     │
├─────────────────────────────────────┤
│     ⏮️    ⏪    ▶️    ⏩    ⏭️      │ ← 第一行：主要控制
│                                     │
│    🔁      ⏹️      ⛶              │ ← 第二行：辅助控制
│   模式    停止   全屏(突出)          │
└─────────────────────────────────────┘
```

### 按钮分布

#### 第一行：主要播放控制（大按钮）
- **⏮️ 上一曲** - 跳到上一首
- **⏪ 快退** - 后退10秒
- **▶️/⏸️ 播放/暂停** - 主控制按钮（浮动按钮）
- **⏩ 快进** - 前进10秒
- **⏭️ 下一曲** - 跳到下一首

#### 第二行：辅助功能（较小按钮）
- **🔁 播放模式** - 切换循环/随机模式（左侧）
- **⏹️ 停止** - 停止播放（中间）
- **⛶ 全屏** - 进入全屏模式（右侧，**突出显示**）

---

## ✨ 全屏按钮特点

### 视觉突出
- **更大的图标**：28px（其他按钮20px）
- **主题色高亮**：使用应用主色调
- **位置固定**：始终在第二行最右侧

### 如何识别
寻找这个图标：**⛶** （四个角向外的箭头）

提示文字：**"全屏播放"**

---

## 🎯 使用方法

### 步骤 1：确保正在播放
底部控制栏只有在有文件播放时才会显示

### 步骤 2：找到全屏按钮
在底部控制栏的**第二行最右侧**

### 步骤 3：点击进入全屏
点击后会自动：
- ✅ 切换到横屏
- ✅ 隐藏系统UI
- ✅ 显示大字体计时器
- ✅ 启用防锁屏

### 步骤 4：退出全屏
在全屏界面底部点击 **"退出全屏"** 按钮

---

## 🔧 为什么这样设计？

### 问题
原来的一行布局有 8 个按钮，在小屏幕设备上会：
- ❌ 按钮过于拥挤
- ❌ 难以准确点击
- ❌ 图标变小不易识别

### 解决方案
采用两行布局：
- ✅ 主要控制在第一行（常用操作）
- ✅ 辅助控制在第二行（次要操作）
- ✅ 每个按钮有足够空间
- ✅ 全屏按钮突出显示

## 💡 提示

### 如果还是看不到全屏按钮

1. **确认有文件在播放**
   - 底部控制栏只在播放时显示
   - 如果没有播放，先选择一个文件播放

2. **热重载应用**
   ```bash
   # 在运行 flutter run 的终端中按 'r'
   # 或者完全重启应用
   flutter run
   ```

3. **检查屏幕尺寸**
   - 确保设备屏幕足够显示两行按钮
   - 小屏幕设备可能需要滚动才能看到第二行

4. **查看日志**
   ```bash
   flutter run --verbose
   ```
   检查是否有任何错误信息

## ❓ 常见问题

### Q: 为什么全屏按钮在第二行？
A: 为了给主要播放控制（播放/暂停、快进/快退）更多空间，同时让全屏按钮更突出。

### Q: 可以把全屏按钮放在第一行吗？
A: 可以，但会导致按钮拥挤。建议保持当前设计。

### Q: 全屏按钮太小了怎么办？
A: 可以调整 `iconSize` 参数，从 28 增加到 32 或更大。

### Q: 为什么我的设备上看不到第二行？
A: 可能是屏幕太小或分辨率问题。尝试：
- 滚动查看是否有更多内容
- 检查应用是否正确编译
- 重启应用
